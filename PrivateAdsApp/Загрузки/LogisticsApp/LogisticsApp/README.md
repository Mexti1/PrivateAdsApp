# Logistics App - Мобильное приложение для транспортно-логистической компании

## Описание проекта

Мобильное приложение для управления услугами транспортно-логистической компании, разработанное на Kotlin с использованием Android Jetpack Compose.

## Основные возможности

### 1. Управление заказами
- Просмотр списка всех заказов
- Фильтрация заказов по статусу
- Детальная информация о каждом заказе
- Создание новых заказов
- Обновление статуса заказов

### 2. Дашборд и статистика
- Общая статистика по заказам
- Количество заказов по статусам
- Общий доход
- Быстрый доступ к основным функциям

### 3. Отслеживание грузов
- Поиск груза по трек-номеру
- Текущее местоположение груза
- История перемещения
- Ожидаемое время прибытия

### 4. Информация о заказах
- Данные клиента (имя, телефон)
- Маршрут доставки (откуда - куда)
- Информация о грузе (описание, вес, объем)
- Данные о водителе и транспорте
- Даты создания и доставки
- Стоимость услуг

## Технологический стек

- **Язык**: Kotlin
- **UI Framework**: Jetpack Compose
- **Архитектура**: MVVM (Model-View-ViewModel)
- **Навигация**: Navigation Compose
- **Управление состоянием**: StateFlow, Compose State
- **Асинхронность**: Kotlin Coroutines
- **Dependency Injection**: Manual DI (можно добавить Hilt/Koin)

## Модели данных

### Order (Заказ)
- ID заказа
- Номер заказа
- Информация о клиенте
- Адреса загрузки и доставки
- Описание груза, вес, объем
- Статус заказа
- Даты создания и доставки
- Цена
- Информация о водителе и транспорте
- Трек-номер

### OrderStatus (Статусы заказа)
- PENDING - Ожидает обработки
- CONFIRMED - Подтвержден
- ASSIGNED - Назначен водитель
- IN_TRANSIT - В пути
- DELIVERED - Доставлен
- CANCELLED - Отменен

### TrackingInfo (Информация для отслеживания)
- Текущее местоположение
- Время последнего обновления
- Ожидаемое время прибытия
- История статусов

## Запуск приложения

### Требования
- Android Studio Hedgehog (2023.1.1) или новее
- Kotlin 1.9.0 или новее
- Android SDK 24 (Android 7.0) или выше
- JDK 8 или новее

### Шаги запуска

1. Откройте проект в Android Studio
2. Синхронизируйте проект с Gradle файлами
3. Подключите Android устройство или запустите эмулятор
4. Нажмите "Run" или используйте Shift+F10

## Архитектура MVVM

Приложение использует архитектуру MVVM:

- **Model** - модели данных и бизнес-логика (data/model, data/repository)
- **View** - UI компоненты на Jetpack Compose (ui/*)
- **ViewModel** - управление состоянием и связь между View и Model

## Управление состоянием

- StateFlow для реактивного управления состоянием
- Compose State для локального состояния UI
- Coroutines для асинхронных операций
